MIGO User Guide

Connecting

Turn on MIGO, then tap the Connect button on the left. Select "Migo_Web_Bot_v4", "Migo_Web_Bot_v3", or "Migo_Web_Bot_v2" from the BLE pairing popup list.

Coding Basics

Drag blocks from the left menu into the workspace. Your code must be snapped inside the Setup block (runs once) or the Loop block (runs forever) to work.

Hardware Plugs

MIGO uses simple Plugs instead of complicated pin numbers. Connect your wires to these pins on the ESP32 board:

  • Plug A (Pin 34): Use for sensors (Buttons, Knobs, Light Sensors).
  • Plug B (Pin 35): Use for sensors.
  • Plug C (Pin 32): Use for sensors.
  • Plug D (Pin 14): Use for outputs (LEDs, Buzzers, Servos).
  • 0.96 OLED LCD: Connect to I2C pins (e.g., SDA Pin 21, SCL Pin 22).
  • Ultrasonic: Automatically uses Trigger (Pin 13) and Echo (Pin 12).

Comprehensive Blocks Guide

Events

  • Setup Block (event_setup): Runs once at the start of your program. Use this to set up pin configurations, turn on a default light, or initialize variables.
  • Loop Forever Block (event_loop): Runs code inside it over and over in an endless cycle. Perfect for checking sensor values and running motors continuously.

Movement & Sound

  • Move [Dir] at Speed [SPD] (move_robot): Drives MIGO Forward, Backward, Spin Left, or Spin Right at speed 0 to 255.
  • Stop Moving (stop_robot): Cuts off power to the drive motors, stopping MIGO immediately.
  • Wait [Time] ms (wait_time): Pauses the program for a specific number of milliseconds (e.g., 1000 ms is 1 second). [Inventor Mode]
  • Wait [Time] seconds (wait_time_s): Pauses the program for a specific number of seconds. [Explorer Mode]
  • Play Tone on [Plug] Freq [Hz] for [ms] (buzzer_tone): Plays a sound at a custom frequency (in Hz) on Plug D or C for a set duration.
  • Turn Servo on [Plug] to [Percentage] % (servo_motor): Rotates a servo motor connected to Plug D or C to a percentage (0% to 100%, corresponding to 45° to 135°).

Sensors (Input)

  • Distance from Ultrasonic (sensor_ultrasonic): Returns the distance to the nearest object in front of the sensor (in centimeters).
  • Is IR Sensor on [Plug] seeing a line? (sensor_ir): Returns true if the infrared sensor on Plug A, B, or C detects a dark line.
  • Is Button on [Plug] pressed? (sensor_button): Returns true if the push button on Plug A, B, or C is pressed.
  • Get Knob Level on [Plug] (sensor_knob): Reads the potentiometer on Plug A, B, or C as a percentage (0% to 100%).
  • Get Light Level on [Plug] (sensor_ldr): Reads the light sensor (LDR) on Plug A, B, or C as a brightness percentage (0% to 100%).
  • Read Digital Value on [Plug] (read_digital): Low-level digital read from Plug A, B, or C (returns 0 or 1).
  • Read Digital on Pin [Pin] (read_digital_custom): Low-level digital read from any custom GPIO pin (0-39) on the ESP32.
  • Read Analog Value on [Plug] (read_analog): Low-level raw analog read from Plug A, B, or C (returns 0 to 4095).
  • Read Analog on Pin [Pin] (read_analog_custom): Low-level raw analog read from any custom ADC pin (0-39) on the ESP32.

Hardware Control (Output)

  • Set [Plug] to [Mode] (setup_pin): Configures Plug A, B, C, or D pin mode as Input (read) or Output (write).
  • Set Custom Pin [Pin] to [Mode] (setup_pin_custom): Configures a custom GPIO pin (0-39) as Input or Output mode.
  • Write Digital [ON/OFF] to [Plug] (digital_write): Outputs High (ON) or Low (OFF) voltage to Plug D or C.
  • Write Digital [ON/OFF] to Pin [Pin] (digital_write_custom): Outputs High or Low voltage to a custom GPIO pin (0-39).
  • Write Analog [Power] to [Plug] (analog_write): Outputs variable power (PWM 0 to 255) to Plug D or C.
  • Write Analog [Power] to Pin [Pin] (analog_write_custom): Outputs variable power (PWM 0 to 255) to a custom GPIO pin (0-39).
  • Turn Built-in LED [ON/OFF] (control_led): Controls the blue onboard status LED (GPIO Pin 2). [Inventor Mode]
  • Turn Light [ON/OFF] (control_led_explorer): Controls the built-in status light. [Explorer Mode]
  • Init OLED Display SDA [SDA] SCL [SCL] (oled_init): Set up the 0.96 OLED display using SDA and SCL pins.
  • Clear OLED Display (oled_clear): Clears the contents on the OLED screen buffer.
  • Print on OLED at X [X] Y [Y] Size [Size]: [Value] (oled_print): Prints a value or message on the screen at specified X, Y coordinates and font size.
  • Show OLED Display (oled_show): Updates the physical screen with the current display buffer contents.

Logic & Math

  • If [Condition] Do (controls_if): Runs internal code only if the condition is true. Click the gear icon to add else/else-if branches.
  • Compare [A] [= / < / >] [B] (logic_compare): Checks if two numbers/values are equal, unequal, greater, or less.
  • Logic [A] [AND / OR] [B] (logic_operation): Combines two logic statements (AND requires both true; OR requires at least one true).
  • Number Input (math_number): Input box to enter numbers.
  • Arithmetic [A] [+ / - / * / /] [B] (math_arithmetic): Computes basic math (addition, subtraction, multiplication, division).
  • Text Input (text): Input box to enter text strings.
  • Repeat [Times] times (control_repeat): Repeats the blocks inside a specific number of times.
  • Set variable [Name] to [Value] (variable_set): Creates a variable and stores a value inside it.
  • variable [Name] (variable_get): Retrieves the value currently stored in a variable.

Functions & Console

  • Define function [Name] (def_func): Combines a group of blocks into a single reusable command.
  • Call function [Name] (call_func): Runs the group of blocks defined under that function name.
  • Print [Value] to monitor (print_serial): Prints a value or message to the Serial Monitor console. [Inventor Mode]
  • Say [Value] (print_serial_explorer): Prints a custom message to the console. [Explorer Mode]
